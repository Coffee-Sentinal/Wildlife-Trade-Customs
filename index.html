Wildlife HS Trade Intelligence Dashboard

Prototype for screening live wildlife trade using HS codes, CITES records, open trade flows, shipment-level uploads, and entity enrichment. Open trade APIs provide aggregate flows. Exporter/importer names require shipment-level records, permits, public manifests, CITES documents, or uploaded intelligence datasets.

Important investigative limit: HS trade-flow APIs will usually return country-pair totals by code, date, value, weight, quantity, and sometimes port. They normally do not reveal exporter/importer entities. This dashboard therefore treats entity-level results as higher-value records from CITES documents, public manifests, permit files, FOIA/public-record data, paid exports you are allowed to use, or uploaded case intelligence.

Recommended investigative logic
  1. Start with a country of export and one or more wildlife HS codes.
  2. Pull aggregate HS flows from Comtrade/WITS, Eurostat Comext, or national APIs.
  3. Overlay CITES Trade Database records by exporter, importer, taxon, source, purpose, term, and year.
  4. Flag mismatches: HS flow exists but no equivalent CITES live-trade record, unexpected quantities, odd origin/destination, or known laundering/transit route.
  5. Attach entity-level rows only from permits, public manifests, shipment records, CITES docs, seizure docs, registry records, or uploaded intelligence.
  6. Enrich entity names with company registries, OpenCorporates, GLEIF, websites, expo vendor lists, online accounts, and known phone/address pivots.
  7. Score the lead as an intelligence hypothesis, not proof. Produce a record with source, confidence, and next verification step.
Minimal backend proxy draft
Use a backend proxy so API keys are not exposed in the browser and to avoid CORS problems. A static HTML page can call these routes. 
// Example Express-style route shape

GET /api/comtrade?reporterCode=360&partnerCode=410&hsCode=010620&flowCode=X&yearFrom=2023&yearTo=2026
  -> calls UN Comtrade API with server-side subscription key
  -> returns normalized rows:
     {source, exporter_country, importer_country, hs_code, period, value_usd, net_weight_kg, quantity}

GET /api/census?direction=exports&hsCode=010620&countryCode=KS&year=2025&month=06
  -> calls U.S. Census International Trade API with server-side key
  -> returns normalized rows, including port/district where available

POST /api/cites-local
  body: {exporter, importer, yearFrom, yearTo, terms:["live"], taxonGroups:["Reptilia"]}
  -> queries local imported CITES Trade Database table

GET /api/opencorporates?q=ENTITY_NAME&jurisdiction=th
  -> returns candidate legal entities and registry metadata

GET /api/speciesplus?name=Lanthanotus%20borneensis
  -> returns CITES appendix, EU annex, quotas, suspensions, range states
